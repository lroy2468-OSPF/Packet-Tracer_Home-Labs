**DHCP Configuration**



**## Overview ##**



This lab demonstrates

-- DHCP

-- End host configuration

-- Trouble shooting

-- Network Design



**## Topology ##**



\- Lab 1 -

* 1 Router
* 2 Switches
* 2 Laptops



\- IP Addressing -

* User1: 192.168.200.0/24
* Gateway: 192.168.200.1



* User2: 192.168.0.0/24
* Gateway: 192.168.0.1



\- Lab 2 -

* 1 Router
* 3 Switches
* 5 Laptops



\- IP Addressing -

* ADMIN: 192.168.1.0/24
* Gateway: 192.168.1.1



* USERS: 192.168.2.0/24
* Gateway: 192.168.2.1



* ACCOUNTING: 192.168.3.0/24
* Gateway: 192.168.3.1



\- Lab 3 -

* 3 Routers
* 3 Switches
* 5 Laptops



\- IP Addressing -

* ENGINEERING: 192.168.2.0/24
* Gateway: 192.168.2.1



* ADMIN: 192.168.1.0/24
* Gateway: 192.168.1.1



* EXECUTIVE: 192.168.3.0/24
* Gateway: 192.168.3.1



* P2P Links: 10.0.0.0/30, 10.0.2.0/30, 10.0.3.0/30





**## Notes \& Skills Demonstrated ##**



-- DHCP Configuration

-- Verification

-- Documentation

-- Troubleshooting

-- Network Design

-- Layer 3 routing

-- Dynamic Host Configuration Protocol was used to assign IP addresses to end host while ensuring network communication. The routing protocol use on Lab 3 was OSPF. These labs also entailed laptop configuration and verification to ensure network stability and reliability.
